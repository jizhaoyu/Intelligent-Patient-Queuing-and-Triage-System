# 三端说明

本文说明当前系统的三类前端入口、使用对象、典型功能和推荐账号。

## 1. 管理后台端

- 入口地址：`http://localhost:5173/login`
- 路由前缀：`/admin/*`
- 使用对象：医院管理人员、运营人员、规则维护人员

### 主要用途

- 查看全局运营数据
- 管理患者基础信息
- 查看候诊队列整体情况
- 维护分诊规则

### 典型页面

- 运营总览
- 患者管理
- 候诊队列
- 分诊规则

### 推荐账号

- `admin`

---

## 2. 工作台端

- 入口地址：`http://localhost:5174/login`
- 路由前缀：`/workstation/*`
- 使用对象：导诊台、分诊护士、诊室医生

### 主要用途

- 查询患者
- 创建就诊记录
- 进行分诊评估
- 执行诊室叫号、重呼、过号、完成接诊

### 典型页面

- 患者查询
- 就诊建档
- 分诊评估
- 诊室叫号

### 推荐账号

- `triage.nurse`
- `guide.desk`
- `doctor.zhang`

---

## 3. 大屏端

- 入口地址：`http://localhost:5175/login`
- 路由前缀：`/screen/*`
- 使用对象：候诊区屏幕、诊室门口屏幕、现场导视展示设备

### 主要用途

- 展示科室候诊信息
- 展示诊室当前叫号信息
- 提示患者排队秩序和候诊状态

### 典型页面

- 科室候诊大屏：`/screen/dept/:deptId`
- 诊室叫号屏：`/screen/room/:roomId`

### 当前库中推荐账号

根据当前数据库绑定关系：

- 科室大屏推荐：`triage.nurse`
  - 绑定 `dept_id=1`
- 诊室大屏推荐：`doctor.zhang`
  - 绑定 `dept_id=1`
  - 绑定 `room_id=1`
- 另一个可用于科室展示的账号：`guide.desk`
  - 绑定 `dept_id=4`

### 不建议账号

- `admin`
  - 当前未绑定 `dept_id`
  - 当前未绑定 `room_id`
  - 不适合作为大屏专用账号

---

## 4. 三端职责总结

| 端 | 作用 | 面向对象 |
|---|---|---|
| 管理后台端 | 管规则、看全局、做运营治理 | 管理人员 |
| 工作台端 | 办业务、做操作、处理接诊流程 | 导诊台、分诊护士、医生 |
| 大屏端 | 做展示、做导视、给患者看 | 候诊区和诊室外屏幕 |
