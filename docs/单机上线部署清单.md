# 患者智能排队分诊系统单机上线部署清单

> 适用场景：1 台 Linux 服务器，使用 MySQL + Redis + Spring Boot JAR + Nginx 部署，不使用 Docker。

## 1. 部署架构

单机部署推荐结构：

- Nginx：托管前端静态资源，并反向代理 `/api`
- Spring Boot：运行后端 JAR，默认端口 `8080`
- MySQL：业务数据库
- Redis：队列与缓存依赖

建议服务器配置：

- 2 核 CPU
- 4 GB 内存
- 40 GB 磁盘
- Ubuntu 22.04 / Rocky Linux 8+ / CentOS 7+

建议开放端口：

- `80`：HTTP
- `443`：HTTPS（如配 SSL）

建议仅本机或内网访问：

- `8080`：后端
- `3306`：MySQL
- `6379`：Redis

## 3. MySQL 配置

### 3.1 创建数据库

```sql
CREATE DATABASE triage_queue DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

### 3.2 创建业务账号

```sql
CREATE USER 'triage'@'localhost' IDENTIFIED BY 'StrongPassword_123!';
GRANT ALL PRIVILEGES ON triage_queue.* TO 'triage'@'localhost';
FLUSH PRIVILEGES;
```

如需远程连接，可将 `localhost` 改为指定 IP 或 `%`，但不建议直接暴露公网。

### 3.3 导入初始化脚本

在项目根目录执行：

```bash
mysql -utriage -p triage_queue < src/main/resources/db/schema.sql
mysql -utriage -p triage_queue < src/main/resources/db/data.sql
```

说明：

- `schema.sql`：初始化表结构
- `data.sql`：初始化演示数据

如果只想建表，不导入演示数据，只执行 `schema.sql` 即可。

## 4. Redis 配置

### 4.1 启动 Redis

Ubuntu / Debian：

```bash
sudo systemctl enable --now redis-server
```

CentOS / Rocky：

```bash
sudo systemctl enable --now redis
```

### 4.2 推荐安全配置

Redis 配置文件常见位置：

- `/etc/redis/redis.conf`
- `/etc/redis.conf`

建议至少配置：

```conf
bind 127.0.0.1
protected-mode yes
```

重启 Redis：

```bash
sudo systemctl restart redis-server
```

验证：

```bash
redis-cli ping
```

返回 `PONG` 表示正常。

> 当前项目配置里未显式声明 Redis 密码项，单机部署建议 Redis 仅监听本机，不直接暴露公网。

## 5. 后端部署

### 5.1 打包后端

在项目根目录执行：

```bash
mvn clean package -DskipTests
```

生成产物：

```bash
target/triage-queue-0.0.1-SNAPSHOT.jar
```

### 5.2 准备环境变量

建议至少配置以下环境变量：

```bash
export DB_URL='jdbc:mysql://127.0.0.1:3306/triage_queue?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai'
export DB_USERNAME='triage'
export DB_PASSWORD='StrongPassword_123!'
export REDIS_HOST='127.0.0.1'
export REDIS_PORT='6379'
export REDIS_DB='0'
export JWT_SECRET='PleaseChangeToYourOwn32CharsOrLongerSecret'
export AI_ENABLED='false'
```

如需启用 AI，再额外配置：

```bash
export AI_ENABLED='true'
export AI_PROVIDER='moonshot'
export AI_MODEL='kimi-k2.5'
export MOONSHOT_BASE_URL='https://api.moonshot.cn/v1'
export MOONSHOT_API_KEY='替换为你自己的密钥'
```

> 不要直接使用仓库中已有的明文 AI Key。上线前应立即更换，并统一改为环境变量注入。

### 5.3 手动启动后端

```bash
nohup java -jar target/triage-queue-0.0.1-SNAPSHOT.jar > logs/backend.log 2>&1 &
```

如 `logs` 目录不存在，请先创建：

```bash
mkdir -p logs
```

### 5.4 检查后端是否启动成功

```bash
curl http://127.0.0.1:8080/actuator/health
```

可选检查：

- Swagger UI：`http://127.0.0.1:8080/swagger-ui.html`
- Knife4j：`http://127.0.0.1:8080/doc.html`

## 6. systemd 托管后端（推荐）

创建文件：

```bash
sudo vi /etc/systemd/system/triage-queue.service
```

写入内容：

```ini
[Unit]
Description=Triage Queue Spring Boot App
After=network.target mysql.service redis-server.service

[Service]
Type=simple
User=root
WorkingDirectory=/opt/triage-queue
Environment=DB_URL=jdbc:mysql://127.0.0.1:3306/triage_queue?useUnicode=true&characterEncoding=UTF-8&serverTimezone=Asia/Shanghai
Environment=DB_USERNAME=triage
Environment=DB_PASSWORD=StrongPassword_123!
Environment=REDIS_HOST=127.0.0.1
Environment=REDIS_PORT=6379
Environment=REDIS_DB=0
Environment=JWT_SECRET=PleaseChangeToYourOwn32CharsOrLongerSecret
Environment=AI_ENABLED=false
ExecStart=/usr/bin/java -jar /opt/triage-queue/target/triage-queue-0.0.1-SNAPSHOT.jar
SuccessExitStatus=143
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动与开机自启：

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now triage-queue
sudo systemctl status triage-queue
```

查看日志：

```bash
journalctl -u triage-queue -f
```

## 7. 前端打包与发布

前端共有 4 个入口：

- 管理后台：admin
- 工作台：workstation
- 大屏：screen
- 患者端 / 自助机：kiosk

### 7.1 安装依赖

```bash
npm --prefix web install
```

### 7.2 分别构建四个前端入口

```bash
npm --prefix web run build:admin
cp -r web/dist /var/www/triage-admin

npm --prefix web run build:workstation
cp -r web/dist /var/www/triage-workstation

npm --prefix web run build:screen
cp -r web/dist /var/www/triage-screen

npm --prefix web run build:kiosk
cp -r web/dist /var/www/triage-kiosk
```

> 注意：当前 Vite 配置默认都输出到 `web/dist`，每次构建会覆盖上一次结果，因此必须在每次构建后立即拷贝到独立目录。

如果目标目录不存在，可先创建：

```bash
sudo mkdir -p /var/www/triage-admin
sudo mkdir -p /var/www/triage-workstation
sudo mkdir -p /var/www/triage-screen
sudo mkdir -p /var/www/triage-kiosk
```

## 8. Nginx 配置示例

推荐使用 4 个子域名分别承载四个前端入口：

- `admin.example.com`
- `work.example.com`
- `screen.example.com`
- `kiosk.example.com`

以下示例都将 `/api/` 反向代理到本机后端 `8080`。

### 8.1 管理后台

```nginx
server {
    listen 80;
    server_name admin.example.com;

    root /var/www/triage-admin;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 8.2 工作台

## 11. 上线前注意事项

### 11.1 必须修改 JWT 密钥

不要使用默认 `JWT_SECRET` 占位值，必须替换为生产环境专用强密钥。

### 11.2 MySQL / Redis 不要直接暴露公网

建议仅本机或内网访问。

### 11.3 AI 配置改为环境变量注入

不要继续使用仓库里的明文配置。

### 11.4 数据迁移脚本仍建议补齐

当前项目更适合初始化环境。若用于已有数据库环境，建议补正式迁移脚本。
